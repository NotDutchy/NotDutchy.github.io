Project Remote Healthcare

For this project I had to create our own server and client system. This was all done in C#. On the left are some screenshots of the end results of the GUI's. The doctor and patient are able to talk to each other via the chat. The goal was to allow the patient to do remote excersises on a bike and send over the data to the doctor. To make the experience more immersive we also created a VR world. The last screenshot is a blank VR world that we edited into something more realistic. Sadly I can't find any old screenshot of our world, so I can't show that off.

Concepts

My contribution

  • Programming in C#
    As this was my first project in C#, it was easy to grasp because it is rather similar to Java. We needed to setup our own server and client system to be able to commmunicate with the doctor or patients.
  • Network protocol
    For this project we needed to create our own network protocol to communicate between our clients and our server.
  • MVVM
    During this project I learned another design pattern called MVVM.
  • MVVM GUI
    One of my jobs during this project was figuring out how to make the GUI's. We used the MVVM pattern for our GUI structure. This makes it easy to separate logic from data objects. This way data can be edited without having to acces the GUI class itself.
  • VR Engine
    Another part I did during this project was figuring out how to change the VR world in the engine.
